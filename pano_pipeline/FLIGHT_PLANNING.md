# Antigravity A1 8K 全景无人机 3DGS 重建飞行规划

> **背景**：本文档基于 scene_021–028 8 段实拍视频的 SfM/3DGS 重建经验整理。其中 021、022、023、025 成功重建，026、027、028 失败。本文档分析失败原因，并提出针对 Antigravity A1（8K 360° 双鱼眼无人机）的可重建飞行规划。

---

## 一、失败案例的根本原因分析

### 1.1 scene_026（垂直爬升型）

**视频内容**：无人机从近地面快速垂直上升到 200m+ 高空再飞越市区。
**问题**：
- 前段：低空近距离物体（建筑、停车场）
- 后段：高空俯瞰大范围地形
- **同一物体在前后段以完全不同尺度出现**，特征描述子无法匹配。
- 纯垂直运动使地面特征点的视差几乎为零 → 三角化失败。

### 1.2 scene_027（起飞→城市飞越→降落型）

**视频内容**：地面起飞 → 高空飞越河流城市 → 在另一地点降落。
**问题**：
- 起飞段（地面停车场，能看到摄影师）
- 中段（数百米高空俯视城市，全新内容）
- 降落段（不同的地面停车场）
- **三段内容完全不连续**，COLMAP 找不到一对能贯穿全场景的「初始图像对」（initial image pair），陷入"Discarding reconstruction due to bad initial pair"循环。

### 1.3 scene_028（观景台→山村航拍→观景台型）

**视频内容**：从观景台木台上起飞 → 山村+梯田航拍 → 回到观景台。
**问题**：与 027 同类。

### 1.4 通用失败模式

```
失败模式: 起点近景 → 中段远景 → 终点近景  (尺度断裂)
         |                                  |
         不可重建为同一场景，COLMAP 必然失败
```

```
成功模式: 持续匀速移动，所有帧相对邻帧位移小、重叠多、尺度一致
         |
         021/022/023/025 都属于此类
```

## 二、3DGS 重建对飞行轨迹的硬性要求

| 维度 | 要求 | 失败值 |
|---|---|---|
| **帧间重叠率** | ≥ 70% | < 50% |
| **相邻帧基线（平移）** | 0.5–3 m | 0（悬停）或 ≥ 10m |
| **三角化角度** | 每个 3D 点至少从 3 个相机看到，夹角 ≥ 5° | 纯前向运动时 ≈ 0° |
| **尺度一致性** | 同一物体最大/最小可见尺寸比 ≤ 3× | 起飞落地时可达 100× |
| **覆盖完整性** | 目标的每个面至少被 3–5 个不同视角拍到 | 只飞过一次的面 |

**结论**：拍摄全程必须保持「位移匀速、视角连续、尺度近似一致」。**严禁起飞和降落画面混入数据**。

## 二·五、定量参数计算（从相机光学 + 视差物理推导）

上面是定性要求；下面把**距离 / 速度 / 基线 / 航线间距**算成公式。基准是 8K equirect 的角分辨率：

```
IFOV = 360° / 7680 = 0.047°/px = 0.82 mrad/px      （标称）
有效 IFOV ≈ 0.82 × 2 ≈ 1.6 mrad/px                 （鱼眼 MTF + 拼接损失，工程取 2×）
```

**1) 离景物多远 —— 由细节 GSD 决定**
```
GSD(cm) = D(m) × 有效IFOV × 100 ≈ 0.16 × D
```
| 目标细节 | 需 GSD | 最大距离 D |
|---|---|---|
| 厘米级纹理 | 1 cm | ≤ 6 m |
| 建筑结构 | 2 cm | ≤ 12 m |
| 粗体块/地形 | 5 cm | ≤ 31 m |

→ 想「看清原貌」**D ≈ 8–20 m**（我们失败的数据是几十米高空俯拍 → GSD 太粗）。

**2) 视差 —— 由深度精度决定（之前缺的关键）**
```
σ_Z/Z ≈ (σ_px · IFOV) / (B/D),  σ_px≈0.3px
要 σ_Z/Z ≤ 2% 且抗噪有余量 → 三角化角 θ = 5°–15°
关键帧基线  B = tan(θ) · D ≈ (0.1 – 0.25) × D
```
| D | 关键帧基线 B (θ=12°) |
|---|---|
| 10 m | 2.1 m |
| 20 m | 4.3 m |
| 30 m | 6.4 m |

→ 单次穿越沿航向 **B/D≈0 → θ≈0 → 深度不可解 → 一团乱麻**（后期无法修复）。

**3) 速度 —— 由运动模糊决定（耦合快门）**
```
v_max = D · IFOV / t_exp      （模糊 < 1 px）
```
快门 ≥ 1/500 s 时模糊几乎不是瓶颈（D=15m → 12 m/s）；真正限速的是「关键帧基线别太大」。

**4) 帧率 —— 满帧采集，按基线抽关键帧**
```
关键帧抽取率 ≈ v / B      （例 D=20m, v=4m/s, B=4.3m → ~0.9 fps）
```

**5) 航线 —— 用上面的数算间距（系统的核心）**

| 模式 | 计算公式 / 取值 |
|---|---|
| 蛇形栅格行间距 | S ≈ B ≈ 0.15–0.25 D（给横向视差） |
| 多高度层 | ≥2 层，高度差 ≈ 0.5 D（给 Z 轴视差） |
| 地标环绕 | R ≈ 1.5–3× 物高，每 8–10° 一帧（≈40 帧/圈），飞 420° |
| 闭环 | 末尾重飞首行 → 闭合位姿图（否则累积漂移，见 480 帧实验） |

**这是一个耦合系统**：靠近(D↓)→GSD 好但行数↑航时↑；飞快(v↑)→省时但基线稀+模糊；
视差(B/D↑)→几何准但航线多。受电池/航时预算约束做权衡，**任何一项缺失（尤其视差）
后期都补不回**。

**一键计算 + 出航点**（`sample_flight_plans/generate_waypoints.py plan`）：
```bash
python generate_waypoints.py plan --center <lat,lon> --size 120x120 \
    --subject-dist 12 --tri-deg 12 --shutter 500 --output myplan
# → 打印 GSD/基线/速度/关键帧率/行间距/双高度，并导出 myplan.kml / .csv 航点
```

## 三、Antigravity A1 8K 设备能力简述

| 参数 | 值 / 说明 |
|---|---|
| 摄像头 | 双鱼眼 360°（每镜头 4K，合成 8K equirectangular） |
| 镜头朝向 | 上下垂直（一上一下，机身两端） |
| 拼接 | 机内/Insta360 Studio 拼接为 equirectangular |
| 续航 | 约 25–35 min（实际可用约 20 min） |
| 飞行速度 | 0.5–15 m/s 可调 |
| GPS / IMU | 有，飞行轨迹可导出 |
| RTK / 厘米级定位 | 标配版无 RTK，建议手动加 |
| 自动航点 | 配套 APP 支持航点规划 |
| 障碍物避障 | 全向（依靠机身传感器） |

### 关键约束
1. **机身朝向无所谓**：360° 相机无视机头方向。飞行规划重点是**位置轨迹**而不是「机头指向哪」。
2. **避免极点遮挡**：双鱼眼接缝在水平面前后，**避免在前后向飞行时长时间盯着同一物体**（拼缝处的物体会模糊）。
3. **GPS 必开**：用于后期 COLMAP geo-registration，大幅加速 SfM 并改善精度。

## 四、推荐飞行模式（按重建目标分）

### 4.1 模式 A：单体目标环绕（雕塑、建筑、车辆）

**适用**：重建一个相对独立的目标。

```
     俯视图                          侧视图
                                
     ↑ 北                          ╱╲ 高层环 (1.5h)
   ╔═══════╗  ←外环             ╱  ╲
   ║       ║                    │ ●  │ 中层环 (1.0h)
   ║  ●●●  ║  (目标)            │目标│
   ║       ║                    ╲  ╱
   ╚═══════╝                     ╲╱  低层环 (0.5h)
   ↘ 速度: 1-2 m/s
```

**参数**：

| 参数 | 推荐值 |
|---|---|
| 飞行高度 | 目标高度 h 的 **0.5h / 1.0h / 1.5h** 三圈 |
| 飞行半径 | 目标尺寸的 1.5–2.5 倍 |
| 飞行速度 | **1–2 m/s** |
| 单圈耗时 | 60–120 秒/圈 |
| 总时长 | 3 圈 × 90 秒 = **4.5 分钟** |
| 帧采样 | 每 1 秒抽 1 张全景 ≈ 270 张 |

**关键检查**：
- 三个高度的圆环必须保持闭合（起点 = 终点）
- 速度恒定，避免悬停或急加速
- 每个高度至少飞 360°+30° 的重叠（420°）以保证首尾匹配

---

### 4.2 模式 B：建筑/广场扫描（推荐用于 026/027/028 这类目标）

**适用**：大范围扫描型场景。

```
   俯视图（网格扫描 + 边缘环绕）
   
   ┌─────────────────────────┐
   │ → → → → → → → → → → →   │  ← 第1层平飞
   │ ← ← ← ← ← ← ← ← ← ← ←   │  ← 蛇形折返
   │ → → → → → → → → → → →   │
   │                          │
   │  ┌────目标区域────┐      │  ← 第2层 (略高)
   │  │     ▓▓▓▓     │      │     重复蛇形
   │  └────────────────┘      │
   └─────────────────────────┘
   
   边界处先飞一圈外环                       
```

**参数**：

| 参数 | 推荐值 |
|---|---|
| 高度 | 比最高建筑高 **20–30m**（保证全景图中地面占多数） |
| 邻行间距 | 高度的 **70%**（重叠 30%）|
| 飞行速度 | **2–3 m/s** |
| 帧采样 | 每 1 秒 1 张 ≈ 0.5 fps |
| 总时长 | 单层 200×100m 区域 ≈ 6–8 分钟 |

**关键检查**：
- 蛇形折返的转弯处放慢到 0.5 m/s 防运动模糊
- 至少做 **两层**（高度差 10–20m）以增加 Z 轴视差
- **不要**起飞后直接进入扫描区 — 让无人机在固定起飞点上升到目标高度后再水平进入

---

### 4.3 模式 C：长条状目标（街道、河道、栈桥）

**适用**：scene_028 那种「沿着木栈桥拍摄」的需求。

```
     沿目标轨迹两侧各飞一次

     ──────目标长条───────       
       ↑ A 侧飞行  
   ↗→→→→→→→→→→→→→→→→→→↗  ← 偏 10m 高度差
                            
   ↘←←←←←←←←←←←←←←←←←←↘  ← 反向 + 10m 间距偏移  
       ↓ B 侧飞行          
     ──────目标长条───────  

   起飞点必须远离扫描区域       
```

**参数**：

| 参数 | 推荐值 |
|---|---|
| 距目标距离 | 5–15m（视目标尺寸） |
| 高度差（两次往返） | 5–10m |
| 飞行速度 | **1–2 m/s** |
| 总时长 | 长度 100m 的栈桥：约 5 分钟 |

## 五、起飞/降落处理（最关键）

```
✗  错误：相机一开机就开始拍 → 数据包含起飞画面
   →  COLMAP 看到 "地面特写" + "高空" 两种尺度，无法对齐

✓  正确：分两段录像
   A. 上升段（不参与重建，不进 SfM）
      - 在远离目标 50m+ 的位置垂直上升
      - 直到达到目标高度
      - 停止录像，停留 5 秒等待稳定
   B. 重建段（参与 SfM/3DGS）
      - 开始新录像
      - 飞入目标区域开始规划好的轨迹
      - 轨迹结束后，先飞出目标区域再停录像
   C. 下降段（不参与）
      - 在远离目标的位置垂直下降
```

**录像分段建议**：
- A1 应当支持手动开停录像。如不能，则在后期剪辑时切掉首尾。
- 切掉的视频长度估算：起飞 + 进入 + 退出 + 降落 ≈ **首尾各 10–20 秒**。
- 后期抽帧时可以用 `-ss 15 -to <end-15>` 跳过这些段。

## 六、Pre-Flight Checklist（飞行前清单）

```
□ 电量 ≥ 80%（避免飞行中途返航）
□ SD 卡剩余空间 ≥ 32GB（8K 视频 ~3GB/分钟）
□ GPS 已锁定 ≥ 12 颗卫星
□ 镜头清洁（手指印会显示在拼缝处）
□ 天气检查
  □ 风速 ≤ 5 m/s（影响飞行稳定性）
  □ 光照均匀，避免强直射阳光产生镜头眩光
  □ 无雨、无雾（雾会显著降低纹理特征）
  □ 黄金时段（日出后 1h / 日落前 1h）光照最佳
□ 目标地图勘察
  □ 在卫星地图上标出目标区域 GPS 边界
  □ 标出起飞点（距目标 ≥ 30m）
  □ 标出禁飞区、高压线、信号塔
□ 航线导入
  □ APP 内确认航线闭合、速度匀速
  □ 高度设置正确（GPS 高度 vs 离地高度）
□ 同步设备时钟（用于多视频时序对齐）
```

## 七、In-Flight 操作要点

| 阶段 | 关键操作 |
|---|---|
| 起飞 | 在远离目标的 GPS 锚点垂直起飞至目标高度 |
| 进入 | 用最低速度（0.5 m/s）水平进入扫描区 |
| 扫描 | 严格匀速，**不要悬停**，**不要倒退** |
| 拐弯 | 速度降到 0.5 m/s，半径不小于 5m |
| 退出 | 水平退出至 GPS 锚点 |
| 降落 | 在锚点垂直下降，避免横风 |

**速度控制**：

```
8K 视频 30 fps，单帧采样间隔 1 秒 → 速度 2 m/s 时，相邻帧位移 2m
建筑特征点至 10m 远 → 2m 位移 = 视差角 ~11° → 适合三角化 ✓

速度 < 0.5 m/s → 视差不足，失败
速度 > 5 m/s → 帧间位移 5m，可能丢失重叠，失败
```

## 八、Post-Flight 数据准备

### 8.1 文件处理

```
原始 .insv (8K 双鱼眼)
   ↓ Insta360 Studio 拼接 (推荐) 或 ffmpeg + v360 (开源备选)
拼接后 equirectangular .mp4
   ↓ ffmpeg 抽帧 1 fps
8K equirectangular JPG 序列
   ↓ pano_to_perspective.py (FOV=120, standard preset)
14 张透视图 / 全景帧
   ↓ COLMAP (sequential_matcher, GPU SIFT)
稀疏 3D 点云 + 相机位姿
   ↓ 3DGS train.py
最终 3D 高斯点云 (.ply)
```

### 8.2 数据规模目标

| 飞行总时长 | 全景帧数 | 透视图数 | 预计训练时间 (单GPU) |
|---|---|---|---|
| 3–5 min | 90–150 | 1260–2100 | 30–60 min |
| 5–10 min | 150–300 | 2100–4200 | 1–2 h |
| 10–15 min | 300–450 | 4200–6300 | 2–4 h |

> 单次飞行不要超过 15 分钟视频用于一个场景。需要更大场景时分段录制再分别重建。

### 8.3 拼接质量验证

抽 3 张相邻全景帧（首/中/尾），人眼检查：

| 检查项 | 通过标准 |
|---|---|
| 拼缝可见性 | 水平接缝（极区附近）模糊带宽度 < 50px |
| 特征点丰富度 | 画面下半部（地面/物体）有清晰纹理（树叶、砖、字） |
| 帧间相似度 | 相邻帧能看到至少 50% 的相同内容 |
| 尺度一致 | 视频首/尾镜头中同一物体大小 ≤ 3× 差异 |

不通过则需要重飞或重新分段处理。

## 九、针对 026/027/028 类失败场景的"补救飞行"建议

如果地点可重访，按以下方式重飞：

### 026 重飞建议（原：垂直爬升）
- **新目标**：选定一个具体目标（如观景台/建筑/小广场）
- **采用模式 A**：在目标高度上做 3 层环绕
- **总时长**：5 分钟

### 027 重飞建议（原：城市飞越）
- **新目标**：选定一个 100×100m 的区域
- **采用模式 B**：网格扫描 + 边缘环绕
- **总时长**：8 分钟

### 028 重飞建议（原：观景台→山村）
- **新目标**：栈桥/观景台作为单独场景
- **采用模式 C**：沿栈桥两侧低空飞行
- **总时长**：5 分钟
- **山村景色** → 另开一段视频做大范围模式 B 重建

## 十、Quick Reference（一页要点）

```
✓ 速度匀速 1–3 m/s
✓ 相邻帧重叠 ≥ 70%
✓ 起飞/降落不入镜（远离目标 50m+ 起降）
✓ 多层环绕 + 多视角 + 多高度
✓ 单段录像 ≤ 15 分钟
✓ GPS 必开
✓ 风速 < 5 m/s, 无雾

✗ 不要悬停（基线为零）
✗ 不要纯垂直运动（视差为零）
✗ 不要混合"地面特写 + 高空俯瞰"
✗ 不要"起飞→飞越→降落"作为同一段
✗ 不要急加减速（动态模糊）
```
